# Getting Help

## Reporting issues

Before opening an issue on
[MPAS-Dev/MPAS-Model](https://github.com/MPAS-Dev/MPAS-Model/issues):
search existing issues, confirm the problem on a current release (v8.4.0+)
or `develop`, and gather environment details.

```markdown
## Description
What's happening, in one or two sentences.

## Reproduction
1. Build with: <compiler/version, flags>
2. Run case: <test case or namelist>
3. Observe: <symptom>

## Expected vs. actual

## Environment
- OS, compiler, MPI, MPAS version (e.g. v8.4.0, commit <sha>)
- Library versions: NetCDF, PnetCDF, PIO

## Additional context
Logs, namelist excerpts.
```

For *usage* questions (rather than bugs), the
[MPAS-Atmosphere Support Forum](http://forum.mmm.ucar.edu/) is more
appropriate than GitHub issues.

## Code of Conduct

The MPAS community welcomes contributors of all backgrounds and experience
levels. Be respectful, be patient, give constructive feedback, assume good
faith. Newcomers especially deserve clear answers and patient review.

Harassment, discrimination, personal attacks, and disclosure of private
information are not tolerated. Reports go to project maintainers via
private channels and are handled confidentially.
